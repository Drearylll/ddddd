## 🎯 待优化的重复逻辑

### 高优先级问题

#### 1. 双重体验路径 ⭐⭐⭐⭐⭐

**问题**：
```
路径 A: / → /feed (Web 轻量版)
路径 B: / → /welcome → ... → /app_main (App 完整版)
```

**现状**：
- 两套独立的用户流程
- `/feed` 包含约 1500 行内容生成逻辑
- `/app_main` 功能相对简单

**建议方案**：

**方案 A：完全统一（推荐）**
```python
# 删除 /feed 路由
# 将所有用户引导到 /welcome

@app.route('/')
def index():
    return redirect(url_for('welcome_premium'))
```

**优点**：
- ✅ 用户体验统一
- ✅ 代码结构清晰
- ✅ 便于维护迭代

**缺点**：
- ❌ 失去轻量级入口

---

**方案 B：保留双路径，明确定位**
```python
# 重命名 /feed 为 /web
@app.route('/web')
def web_experience():
    """Web 轻量体验 - 快速浏览"""

# 首页提供选择
@app.route('/')
def index():
    return render_template('choice.html')
```

**优点**：
- ✅ 满足不同需求
- ✅ 降低门槛

**缺点**：
- ❌ 维护成本高

#### 3. 用户数据管理混乱 ⭐⭐⭐

**当前问题**：
```javascript
// 前端 localStorage
localStorage.setItem('goIn_userName', userName);
localStorage.setItem('goIn_auth_completed', 'true');
```

```python
# 后端 Session
session['user_name'] = user_name
session['onboarding_completed'] = True
```

```python
# 文件存储
save_user_data('preferences', prefs)
```

**三重存储可能导致**：
- 数据不一致
- 同步问题
- 调试困难

**优化建议**：

**统一使用 Session + 后端存储**：
```python
class UserManager:
    @staticmethod
    def get_current_user():
        """获取当前用户完整数据"""
        user_id = get_or_create_user_id()
        user_data = load_user_data(user_id)
        
        # 同步 Session
        for key in session:
            if key.startswith('user_'):
                user_data[key] = session[key]
        
        return user_data
    
    @staticmethod
    def save_data(data):
        """统一保存接口"""
        user_id = get_or_create_user_id()
        save_user_data(user_id, data)
        
        # 同步 Session
        for key, value in data.items():
            if isinstance(value, (str, int, float, bool)):
                session[key] = value
```

## 🎯 最佳实践建议

### 1. 代码组织

**原则**：
- DRY (Don't Repeat Yourself)
- 单一职责
- 关注点分离

**实践**：
```python
# ❌ 坏味道：重复代码
def func_a():
    # 50 行逻辑
    pass

def func_b():
    # 同样的 50 行逻辑
    pass

# ✅ 好做法：提取公共函数
def common_logic():
    # 50 行逻辑
    pass

def func_a():
    return common_logic()

def func_b():
    return common_logic()
```
