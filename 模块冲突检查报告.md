## ⚠️ 三、潜在冲突点分析

### 3.1 双重实现问题

**发现**: App 原型体验有**两套实现方式**

#### 方案 A：使用模板文件（原有）
```python
@app.route('/onboarding') → render_template('onboarding.html')
@app.route('/auth') → render_template('auth.html')
@app.route('/personality_customization') → render_template('personality_customization.html')
@app.route('/app_main') → render_template('app_main.html')
```

#### 方案 B：内联 HTML 字符串（新增）
```python
@app.route('/app') → return '''<html>...'''
@app.route('/app/onboarding') → API 处理
@app.route('/app/customization') → API 处理
@app.route('/app/main') → 重定向到 /app
```

**影响评估**:
- ✅ **无直接冲突**: 两条路径独立（`/onboarding` vs `/app`）
- ⚠️ **功能冗余**: 用户可能混淆应该访问哪个 URL
- ⚠️ **维护成本**: 需要同时维护两套 UI 代码

**建议**: 
1. **保留模板文件方案**（更规范、易维护）
2. **删除内联 HTML 方案**（`/app` 相关路由）
3. **统一入口**: 推荐用户使用 `/onboarding` 作为 App 体验入口
