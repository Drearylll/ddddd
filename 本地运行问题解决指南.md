## ❌ 可能的错误原因

### 错误 1：访问了错误的地址

**错误示例**：
- ❌ `http://127.0.0.1:5000/`（带斜杠）
- ❌ `https://127.0.0.1:5000`（使用了 HTTPS）
- ❌ `http://localhost:5001`（端口错误）

**正确方式**：
- ✅ `http://localhost:5000`
- ✅ `http://127.0.0.1:5000`

### 错误 2：应用未启动

**检查方法**：
```bash
# 查看端口 5000 是否在监听
netstat -ano | findstr :5000
```

**应该看到**：
```
TCP    0.0.0.0:5000    0.0.0.0:0    LISTENING
```

### 错误 3：浏览器缓存问题

**解决方法**：
1. 清除浏览器缓存
2. 使用无痕模式/隐私模式
3. 强制刷新：`Ctrl + Shift + R`

### 错误 4：防火墙阻止

**解决方法**：
1. 检查 Windows 防火墙设置
2. 允许 Python 访问网络
3. 临时关闭防火墙测试

## 🔍 故障排查

### 检查点 1：端口是否被占用

```bash
netstat -ano | findstr :5000
```

**结果分析**：
- 如果看到 `LISTENING` - ✅ 正常
- 如果什么都没看到 - ❌ 应用未启动
- 如果看到其他进程占用 - ❌ 端口冲突

### 检查点 2：应用是否运行

查看运行 Flask 应用的终端窗口：
- 应该看到 "Running on http://127.0.0.1:5000"
- 不应该看到错误信息

### 检查点 3：浏览器控制台

打开浏览器开发者工具（F12），查看：
- Console 标签：是否有错误
- Network 标签：请求状态

## 💡 常见错误及解决

### 错误：Connection refused

**原因**：应用未启动或端口错误

**解决**：
1. 确认应用已启动
2. 检查端口是否为 5000
3. 尝试重启应用

### 错误：404 Not Found

**原因**：路由不存在

**解决**：
1. 访问根路径 `/`
2. 检查 URL 是否正确

### 错误：500 Internal Server Error

**原因**：服务器端错误

**解决**：
1. 查看 Flask 应用日志
2. 检查错误详情
3. 根据错误修复
