# 🌙 未完成感机制

## 📋 功能概述

让部分内容不完全结束，产生轻微的延续感，让用户感觉"好像还会发生点什么"。

---

## 🎯 核心目标

**用户在看到内容后产生一种感觉**：
> "好像还会发生点什么"

---

## ✨ 功能特性

### 1. **未完成句池（20 条）**

#### **轻微行动延续**
- 又往前走了一点。
- 还没决定要不要进去。
- 在门口站了一会儿。
- 可能会再过去。
- 先这样。
- 再看看。
- 再坐一会儿。
- 还不想走。
- 再等等。
- 再待一会儿。

#### **停留、观察**
- 又看了一眼。
- 还是老样子。
- 没什么变化。
- 和之前一样。

#### **轻微犹豫**
- 不太确定。
- 再说吧。
- 下次也可以。
- 改天再来。

---

### 2. **触发概率**

```python
约 20% 概率触发
```

---

### 3. **插入位置**

**文案结尾**，例如：

**原**：
```
坐了一会儿
```

**改**：
```
坐了一会儿
又往前走了一点。
```

---

### 4. **组合方式**

**原文案 + 换行 + 未完成句**

示例：

| 原文案 | 未完成句 | 组合效果 |
|--------|----------|----------|
| 江边有街头艺人唱歌，听了一会儿。 | 还不想走。 | 江边有街头艺人唱歌，听了一会儿。<br>还不想走。 |
| 随便逛了一圈，什么都没买。 | 再看看。 | 随便逛了一圈，什么都没买。<br>再看看。 |
| 坐了一个下午，没人打扰。 | 再坐一会儿。 | 坐了一个下午，没人打扰。<br>再坐一会儿。 |

## 🔧 技术实现

### 1. **未完成句池定义**

```python
UNFINISHED_SENTENCES = [
    # 轻微行动延续
    "又往前走了一点。",
    "还没决定要不要进去。",
    "在门口站了一会儿。",
    "可能会再过去。",
    "先这样。",
    "再看看。",
    "再坐一会儿。",
    "还不想走。",
    "再等等。",
    "再待一会儿。",
    
    # 停留、观察
    "又看了一眼。",
    "还是老样子。",
    "没什么变化。",
    "和之前一样。",
    
    # 轻微犹豫
    "不太确定。",
    "再说吧。",
    "下次也可以。",
    "改天再来。",
]
```

---

### 2. **触发逻辑**

```python
# 未完成感机制：约 20% 概率触发
is_unfinished = random.random() < 0.2  # 20% 概率

if is_unfinished and UNFINISHED_SENTENCES:
    # 从未完成句池中随机选择
    unfinished_sentence = random.choice(UNFINISHED_SENTENCES)
    print(f"[UNFINISHED] Triggered! Adding: {unfinished_sentence}")
else:
    unfinished_sentence = None
```

---

### 3. **文案组合**

```python
# 文案变化处理
text = vary_text_style(selected['text'], gen_count)

# 未完成感机制：在文案结尾添加未完成句
if unfinished_sentence:
    text = text + '\n' + unfinished_sentence

new_post = {
    'time_label': time_label,
    'location': selected['location'],
    'text': text,
    'mood': selected['mood'],
    'is_unfinished': bool(unfinished_sentence),  # 标记是否包含未完成句
}
```

## 🎯 设计原则

### 1. **不要过于明确行动指令**

❌ **错误示例**：
- "我要去下一个地方。"
- "现在出发去那里。"
- "决定去吃饭。"

✅ **正确示例**：
- "又往前走了一点。"
- "再看看。"
- "还不想走。"

---

### 2. **不要出现"我要去"**

❌ **错误示例**：
- "我要去江边。"
- "我要进去看看。"
- "我要再待一会儿。"

✅ **正确示例**：
- "又往前走了一点。"
- "在门口站了一会儿。"
- "再待一会儿。"

---

### 3. **不要解释原因**

❌ **错误示例**：
- "因为累了，所以坐下。"
- "因为下雨，所以离开。"
- "因为无聊，所以走走。"

✅ **正确示例**：
- "坐了一会儿。"
- "又看了一眼。"
- "再看看。"

## 🎭 心理学原理

### 蔡格尼克效应（Zeigarnik Effect）

**定义**：人们对未完成的事情记忆更深刻。

**应用**：
- ✅ 文案不完全结束
- ✅ 产生轻微延续感
- ✅ 让用户感觉"还会发生什么"

---

### 留白美学

**定义**：适当的空白让内容更有想象空间。

**应用**：
- ✅ 不说明下一步
- ✅ 不解释原因
- ✅ 让用户自行想象

## 📝 注意事项

### 1. **概率控制**

- ✅ 20% 概率（约每 5 条出现 1 条）
- ❌ 不要过高（避免审美疲劳）
- ❌ 不要过低（效果不明显）

---

### 2. **文案长度**

- ✅ 短句（2-8 个字）
- ❌ 避免长句（破坏简洁感）
- ❌ 避免复杂句（难以理解）

---

### 3. **情绪匹配**

- ✅ 平静、淡然、随意
- ❌ 激动、兴奋、强烈
- ❌ 避免过于情绪化

## 🎭 哲学思考

### 为什么设计未完成感？

**传统内容**：
```
发生 → 结束 → 下一条
```

**未完成感内容**：
```
发生 → 好像还没结束 → 下一条
```

### 核心差异

| 维度 | 传统 | 未完成感 |
|------|------|----------|
| 结束感 | 强 | ✅ 弱 |
| 延续感 | 无 | ✅ 有 |
| 想象空间 | 小 | ✅ 大 |
| 用户感觉 | "结束了" | ✅ "还会发生" |
