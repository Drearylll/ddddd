## 🌐 公网访问（推荐）

### 方式一：Vercel 部署（最简单，30 秒完成）

**1. 上传到 GitHub**
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/你的用户名/goin.git
git push -u origin main
```

**2. 在 Vercel 部署**
1. 访问 https://vercel.com
2. 点击 "New Project"
3. 选择 "Import Git Repository"
4. 选择你的 goin 仓库
5. 点击 "Deploy"

**3. 获取链接**
- 部署完成后获得：`https://goin-xxx.vercel.app`
- 可直接分享给任何人体验

---

### 方式二：Railway 部署

**1. 安装 CLI**
```bash
npm install -g @railway/cli
```

**2. 登录并部署**
```bash
railway login
railway init
railway up
```

**3. 获取链接**
```bash
railway domain
```

## ✨ 产品特性

### 🎯 核心体验
- ✅ **直接展示内容流** - 没有欢迎页，没有引导页
- ✅ **自动生成初始内容** - 首次访问自动生成 3-5 条内容
- ✅ **AI 自主生活流** - 内容会根据时间间隔自动产生
- ✅ **真实感地点** - 15 个上海浦东具体真实地点
- ✅ **用户隔离** - 每个用户看到独立内容
- ✅ **无登录注册** - 使用隐式 UUID，用户无感知

### 📍 地点示例
- 世纪大道地铁站口
- 陆家嘴滨江步道
- IFC 商场外
- 正大广场门口
- 东方明珠下
- 张江咖啡馆
- 前滩太古里
- 金桥路街边
- 浦东美术馆
- 世纪公园

### ⏰ 时间标签
- 刚刚
- 15 分钟前
- 1 小时前
- 3 小时前

## 🎭 用户体验流程

### 第一次访问
1. 点开链接 → 直接看到 3-5 条内容
2. 内容带时间标签（刚刚 / 15 分钟前 / 1 小时前）
3. 地点真实（世纪大道地铁站口、陆家嘴滨江步道等）
4. 文案有情绪（"也不是非要来。"、"好像不是这里的问题。"）

### 短时间后返回（15 分钟内）
- 不生成新内容或生成 0-1 条
- 保持内容的自然性

### 1 小时后返回
- 生成 1-2 条新内容
- 时间分布在过去 1 小时中

### 5 小时后返回
- 生成 3-5 条新内容
- 时间分布在 5 小时中（±30% 随机浮动）

### 目标效果
用户看到内容时会想：
> "这个地方，我好像真的去过"
> "这里，好像已经在发生一些事情了"

## 数据结构

每条内容包含：
- `time_label`: 时间标签（如"今天早上 7:42"）
- `location`: 真实地点名称
- `text`: 有情绪的文案
- `mood`: 心情标签
