## 🎯 发现的重复逻辑问题

### 问题 1：双重体验路径 ⭐⭐⭐⭐⭐

**现状**：
```
路径 A: / → /feed (Web 轻量版，约 1500 行代码)
路径 B: / → /welcome → /personality_customization → /app_main (App 完整版)
```

**影响**：
- 两套独立的用户流程
- 内容生成逻辑分散在两个地方
- 用户体验不一致

**解决方案**：统一到路径 B（行业顶级设计）

---

### 问题 2：API 端点分散 ⭐⭐⭐

**当前 API**（9 个）：
```python
/save_preferences              # 表单提交
/api/save_avatar_info         # JSON 提交
/api/answer_question          # AI 回答
/api/toggle_agent_mode        # 切换模式
... (共 9 个)
```

**问题**：
- 命名风格不统一
- 缺少文档
- 难以维护

**解决方案**：统一为 RESTful 风格

---

### 问题 3：用户数据管理混乱 ⭐⭐⭐

**三重存储**：
```javascript
// 前端 localStorage
localStorage.setItem('goIn_userName', userName);

// 后端 Session
session['user_name'] = user_name

// 文件存储
save_user_data('preferences', prefs)
```

**问题**：
- 数据同步困难
- 调试复杂
- 可能出现不一致

**解决方案**：统一使用 Session + 后端存储

#### 任务 2：简化路由 ⭐⭐⭐⭐

**方案**：

```python
# 统一用户体验路径
@app.route('/')
def index():
    return redirect(url_for('welcome_premium'))

@app.route('/app_main')
def app_main():
    from services import ContentGenerator
    
    user_data = get_user_data()
    posts = ContentGenerator.generate_ai_life_posts(
        user_data=user_data,
        num_posts=5
    )
    
    return render_template('app_main.html', posts=posts)
```

**删除或注释掉** `/feed` 路由（如果选择完全统一）

#### 任务 3：统一 API 接口 ⭐⭐⭐⭐

**创建统一的 API 路由**：

```python
# routes/api.py
from flask import Blueprint, request, jsonify

api = Blueprint('api', url_prefix='/api')

@api.route('/preferences', methods=['POST'])
def save_preferences():
    """统一的偏好设置保存接口"""
    data = request.get_json()
    
    if not data:
        return jsonify({'success': False, 'message': '缺少数据'}), 400
    
    user_data = get_user_data()
    user_data.update(data)
    save_user_data(user_data)
    
    return jsonify({
        'success': True,
        'message': '偏好已保存'
    })
```
