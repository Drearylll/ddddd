# 地点路径连续性机制说明

## 一、核心目标

让地点之间产生"移动关系"，而不是随机跳跃。

**用户体验目标：**
> "他刚刚是在这附近走了一圈"

---

### 2. 生成逻辑（70/30 法则）

当生成新内容时：

- **70% 概率：** 从当前地点所在区域中选择下一个地点（同区域移动）
- **30% 概率：** 随机跳转到其他区域（跨区跳转）

**示例流程：**

```
第 1 条：IFC 商场外（陆家嘴区域）
   ↓ 70% 概率同区域移动
第 2 条：陆家嘴滨江步道（陆家嘴区域）
   ↓ 70% 概率同区域移动
第 3 条：东方明珠下（陆家嘴区域）
   ↓ 30% 概率跨区跳转
第 4 条：张江咖啡馆（张江区域）
   ↓ 70% 概率同区域移动
第 5 条：金科路商业街（张江区域）
```

## 三、技术实现

### 核心函数：get_next_location()

```python
def get_next_location(last_location=None):
    """
    基于上一个地点生成下一个地点（70% 同区域移动，30% 跨区跳转）
    
    Args:
        last_location: 上一条内容的地点
    
    Returns:
        tuple: (新地点，是否使用移动文案)
    """
    if not last_location:
        # 第一次生成，随机选择地点
        region = random.choice(list(PUDONG_LOCATIONS.keys()))
        location = random.choice(PUDONG_LOCATIONS[region])
        return location, False
    
    # 获取上一个地点所在的区域
    current_region = LOCATION_TO_REGION.get(last_location)
    
    if not current_region:
        # 如果找不到区域，随机选择
        region = random.choice(list(PUDONG_LOCATIONS.keys()))
        location = random.choice(PUDONG_LOCATIONS[region])
        return location, False
    
    # 70% 概率在同区域移动，30% 概率跨区跳转
    if random.random() < 0.7:
        # 同区域移动
        same_region_locations = PUDONG_LOCATIONS[current_region]
        if len(same_region_locations) > 1:
            # 排除当前地点，避免重复
            available_locations = [loc for loc in same_region_locations if loc != last_location]
            if available_locations:
                new_location = random.choice(available_locations)
                print(f"[PATH] Same region move: {last_location} → {new_location} ({current_region})")
                return new_location, True  # 使用移动文案
            else:
                # 区域内只有一个地点，保持原地
                return last_location, False
        else:
            return last_location, False
    else:
        # 跨区跳转
        all_regions = list(PUDONG_LOCATIONS.keys())
        other_regions = [r for r in all_regions if r != current_region]
        new_region = random.choice(other_regions)
        new_location = random.choice(PUDONG_LOCATIONS[new_region])
        print(f"[PATH] Cross region jump: {last_location} → {new_location} ({current_region} → {new_region})")
        return new_location, False  # 不使用移动文案
```

## 八、限制条件

### ❌ 不做的事情

1. **不需要地图 API** - 不计算真实距离
2. **不需要真实坐标** - 只使用逻辑分组
3. **不需要显示路线** - 用户看不到路径
4. **不需要导航功能** - 不是地图应用

---

### ✅ 要做的事情

1. **保持地点真实性** - 所有地点都是真实的
2. **保持文案简洁** - 移动感文案也要简短
3. **保持随机性** - 70/30 是概率，不是固定规则
4. **保持自然** - 不要让用户察觉到算法

---

## 九、设计原则

### 1. 真实世界映射

所有地点必须是真实存在的，且：
- ✅ 可在地图上找到
- ✅ 用户可能真的去过
- ✅ 符合地理逻辑（相邻地点更可能被访问）

---

### 2. 平行自我原则

内容必须表现为"一个可能的用户版本"在持续生活：
- ✅ 地点连续性（一个人在附近活动）
- ✅ 行为合理性（不会瞬间移动）
- ✅ 时间连贯性（时间递进合理）

---

### 3. 不可控体验

用户不能控制或预测路径：
- ✅ 70/30 概率是随机的
- ✅ 用户无法选择目的地
- ✅ 路径是自动生成的
