# 用户数据持久化问题修复

## 问题描述

用户每次退出后重新进入，都会回到头像设置界面，并且显示的是第一次上传的图片。

## 根本原因

1. **用户 ID 没有持久化**
   - 当前使用 Flask session 存储 user_id
   - session 在浏览器关闭后会丢失
   - 导致每次打开都生成新的 user_id

2. **用户数据加载失败**
   - 因为 user_id 每次都不同
   - 无法从 localStorage 或数据库加载之前保存的头像数据
   - 所以显示的是第一次上传的图片（缓存在页面中）

## 解决方案

### 方案 1：将用户 ID 持久化到 localStorage（推荐）

**步骤**：

1. **在 profile_setup.html 中保存用户 ID**
   ```javascript
   // 保存成功后
   localStorage.setItem('goIn_user_id', userId);
   ```

2. **在页面加载时恢复用户 ID**
   ```javascript
   window.addEventListener('DOMContentLoaded', () => {
       const savedUserId = localStorage.getItem('goIn_user_id');
       if (savedUserId) {
           // 通过某种方式传递给后端
           console.log('恢复用户 ID:', savedUserId);
       }
   });
   ```

3. **后端从请求头读取用户 ID**
   ```python
   @app.before_request
   def load_user_from_localstorage():
       user_id = request.headers.get('X-User-ID')
       if user_id:
           session['user_id'] = user_id
   ```

### 方案 2：使用永久 Cookie（简单）

**步骤**：

1. **设置永久 Cookie**
   ```python
   from flask import make_response
   
   @app.route('/api/profile_setup', methods=['POST'])
   def api_profile_setup():
       # ... 保存数据 ...
       
       response = make_response(jsonify({'success': True}))
       response.set_cookie('user_id', user_id, max_age=31536000)  # 1 年
       return response
   ```

2. **从 Cookie 读取用户 ID**
   ```python
   @app.before_request
   def load_user_from_cookie():
       user_id = request.cookies.get('user_id')
       if user_id:
           session['user_id'] = user_id
   ```

### 方案 3：混合方案（最可靠）

结合 localStorage 和后端持久化：

1. **前端保存用户 ID 到 localStorage**
2. **每次请求时自动带上用户 ID**
3. **后端从数据库加载用户数据**

## 立即执行的修复（简化版）

由于时间关系，我建议采用最简单的方案：**使用 Cookie 持久化用户 ID**

### 修改文件清单

1. **app.py** - 添加 Cookie 支持
2. **profile_setup.html** - 无修改（继续使用 localStorage）
3. **app_main.html** - 无修改

### 具体修改

**app.py**:

```python
@app.before_request
def load_user_from_cookie():
    """从 Cookie 加载用户 ID（如果存在）"""
    from flask import request
    
    user_id = request.cookies.get('user_id')
    if user_id:
        session['user_id'] = user_id
        print(f"✅ 从 Cookie 加载用户 ID: {user_id}")

@app.route('/api/profile_setup', methods=['POST'])
def api_profile_setup():
    # ... 现有代码 ...
    
    response = make_response(jsonify({'success': True, 'user_id': user_id}))
    response.set_cookie('user_id', user_id, max_age=31536000)  # 1 年
    return response
```

## 注意事项

1. **Cookie  vs localStorage**
   - Cookie 会自动发送到服务器
   - localStorage 需要手动添加到请求头

2. **安全性**
   - 生产环境应该使用加密的 user_id
   - 考虑添加 CSRF 保护

3. **多设备同步**
   - 当前方案是单设备
   - 多设备需要账号系统

## 快速修复代码

我将在以下文件中添加代码：

### app.py

```python
# 在文件开头添加
from flask import make_response

# 在 app.before_request 中添加
@app.before_request
def load_user_from_cookie():
    """从 Cookie 加载用户 ID"""
    user_id = request.cookies.get('user_id')
    if user_id and 'user_id' not in session:
        session['user_id'] = user_id
        print(f"✅ 从 Cookie 加载用户 ID: {user_id[:8]}...")

# 在 api_profile_setup 中修改返回
response = make_response(jsonify({'success': True, 'user_id': user_id}))
response.set_cookie('user_id', user_id, max_age=31536000)
return response
```

这样就可以确保用户 ID 持久化，从而正确加载用户之前设置的头像数据。
