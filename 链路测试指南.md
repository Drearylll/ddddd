## 🐛 常见问题排查

### 问题 1：访问 `/` 没有跳转

**症状**：显示旧的"选择模式"页面

**原因**：Flask 服务器未重启

**解决方法**：
```bash
# 停止服务器（Ctrl+C）
# 重新启动
python app.py
```

---

### 问题 2：点击"继续"按钮无反应

**症状**：点击后没有任何反应

**排查步骤**：

1. **打开浏览器控制台**（F12）
2. **查看是否有错误**
   ```javascript
   // 应该看到类似输出
   Button clicked
   User name: 小明
   AI name: 小智
   ```

3. **检查 Network 标签**
   - 请求 URL：`/api/save_avatar_info`
   - 请求方法：POST
   - 状态码：200

**可能的问题**：
- ❌ JavaScript 错误 → 查看控制台
- ❌ API 返回 404 → 服务器未重启
- ❌ 网络错误 → 检查 Flask 是否运行

---

### 问题 3：API 返回 404

**症状**：控制台显示
```
POST http://localhost:5000/api/save_avatar_info 404 (NOT FOUND)
```

**原因**：Flask 服务器未加载新代码

**解决方法**：
```bash
# 1. 停止服务器（Ctrl+C）
# 2. 确认代码已保存
# 3. 重新启动
python app.py
```

**验证**：
启动日志中应该看到：
```
✅ 保存用户信息：user_name=xxx, ai_name=xxx
```

---

### 问题 4：跳转后数据丢失

**症状**：到了下一页，但数据没了

**排查**：

1. **检查 localStorage**
   ```javascript
   // 在控制台执行
   console.log(localStorage.getItem('goIn_userName'));
   console.log(localStorage.getItem('goIn_aiName'));
   ```

2. **预期输出**：
   ```
   小明
   小智
   ```

3. **如果为 null**：
   - 说明 localStorage 未正确保存
   - 检查浏览器的隐私模式是否禁用
   - 尝试清除缓存后重试

---

### 问题 5：Logo 无法显示

**症状**：Logo 位置显示一个 Emoji🌆

**原因**：图片文件路径错误或不存在

**检查**：
```bash
# 确认文件存在
dir "c:\Users\hn\Desktop\桌面的东西\GOIN2\static\images\logo.png"
```

**预期输出**：
```
目录: c:\Users\hn\Desktop\桌面的东西\GOIN2\static\images

Mode                 LastSaveDate         Length Name
----                 ------------         ------ ----
-a----        2026/3/20     下午 XX:XX    677176 logo.png
```

**解决**：
- 如果文件不存在，请将 Logo 图片保存到该路径
- 检查文件名是否为 `logo.png`（区分大小写）
