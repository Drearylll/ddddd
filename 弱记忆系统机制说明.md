# 弱记忆系统机制说明

## 一、核心目标

让内容在重复出现时，产生轻微的"来过"的感觉。

**用户体验目标：**
> "这个人，好像记得这里"

---

## 二、实现方式

### 1. 记录历史地点

保存最近 5-10 个已出现地点：

```python
# 获取用户已访问的地点记录
visited_locations = user_data.get('visited_locations', [])

# 更新已访问地点记录
if new_location not in visited_locations:
    visited_locations.append(new_location)
    # 保留最近 20 个地点
    if len(visited_locations) > 20:
        visited_locations = visited_locations[-20:]
```

**实际使用：** 最近 10 个地点（`visited_locations[-10:]`）

---

### 2. 再次出现判断

当新内容地点在历史记录中时：

**触发概率：** 约 30%

```python
# 检查新地点是否已经在历史记录中出现过
is_in_history = new_location in visited_locations
memory_trigger = False

# 如果在历史记录中，30% 概率触发记忆感
if is_in_history and random.random() < 0.3:
    memory_trigger = True
    print(f"[MEMORY] Memory triggered at: {new_location}")
```

**关键点：**
- ✅ 只在地点重复时可能触发
- ✅ 不是每次重复都触发（30% 概率）
- ✅ 保持随机性和自然性

---

### 3. 文案变化

在原句中加入轻微记忆词（8 种记忆片段）：

**记忆片段池：**
```python
MEMORY_FRAGMENTS = [
    "又",      # 又坐了一会儿
    "还是",    # 还是在这里
    "有点熟",  # 有点熟
    "照旧",    # 照旧点了咖啡
    "依旧",    # 依旧安静
    "老位置",  # 老位置
    "熟悉",    # 熟悉的角落
    "再次",    # 再次路过
]
```

**修改规则：**

| 记忆片段 | 修改方式 | 示例 |
|----------|----------|------|
| **又**、**再次** | 句首添加 | "风很大" → "又风很大" |
| **还是**、**依旧**、**照旧** | 句首添加 | "安静" → "还是安静" |
| **有点熟**、**熟悉** | 独立短句 | → "有点熟。" |
| **老位置** | 特殊处理 | → "老位置。" |

**修改逻辑：**
```python
if memory_trigger:
    selected = selected.copy()
    original_text = selected['text']
    
    # 随机选择一个记忆片段
    memory_fragment = random.choice(MEMORY_FRAGMENTS)
    
    # 根据记忆片段类型修改文案
    if memory_fragment in ["又", "再次"]:
        modified_text = f"{memory_fragment}{original_text}"
    elif memory_fragment in ["还是", "依旧", "照旧"]:
        modified_text = f"{memory_fragment}{original_text}"
    elif memory_fragment in ["有点熟", "熟悉"]:
        modified_text = f"{memory_fragment}。"
    elif memory_fragment == "老位置":
        modified_text = "老位置。"
    else:
        modified_text = f"{memory_fragment}{original_text}"
    
    selected['text'] = modified_text
```

---

### 4. 使用规则

#### 不要每次重复都触发

- 地点重复是轻习惯系统的职责（20% 概率）
- 记忆触发是弱记忆系统的职责（30% 概率）
- 两者独立，互不影响

#### 保持随机

- 记忆片段随机选择（8 选 1）
- 触发概率严格控制（30%）
- 不解释、不说明、不刻意

## 八、限制条件

### ❌ 不做的事情

1. **不允许出现明确时间**
   - ❌ "上次来这里..."
   - ❌ "昨天见过..."
   - ✅ "又..."、"还是..."

2. **不允许解释记忆**
   - ❌ "我记得..."
   - ❌ "之前来过..."
   - ✅ "有点熟。"、"老位置。"

3. **不要长句**
   - ❌ "这个地方我好像之前来的时候也看到过类似的场景"
   - ✅ "还是这样。"

---

### ✅ 要做的事情

1. **保持微妙**
   - 轻微的熟悉感
   - 不解释原因
   - 像随口一说

2. **保持随机**
   - 不是每次重复都触发
   - 记忆片段随机选择
   - 修改方式自然

3. **保持简洁**
   - 短句为主
   - 不啰嗦
   - 不刻意

### 2. 真实世界映射

真实的人都有记忆：
- ✅ 会记得常去的地方
- ✅ 有熟悉感
- ✅ 会用微妙的语言表达

---

### 3. 克制表达

不过度渲染：
- ✅ 30% 触发概率（不是 100%）
- ✅ 轻微记忆词（不是长篇大论）
- ✅ 不解释（不说"上次"）

## 十二、后续优化方向

### 1. 记忆强度分级（可选）

根据重复次数调整记忆强度：
- 重复 2 次：轻微记忆（"又..."）
- 重复 5 次：中等记忆（"经常..."）
- 重复 10 次：强烈记忆（"老地方..."）

---

### 2. 记忆累积（可选）

记录某些特殊地点的访问次数：
- "第 10 次来这里" → 特殊文案
- "每周必去" → 习惯强化

---

### 3. 记忆淡化（可选）

长时间不去的地点，记忆逐渐淡化：
- 超过 1 天未去 → 降低触发概率
- 超过 1 周未去 → 不再触发记忆

---

### 4. 记忆组合（可选）

结合多个记忆点：
- 地点 + 行为："又在这里喝咖啡"
- 地点 + 时间："还是这个时间点"
