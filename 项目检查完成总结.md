### 2.2 双重实现问题

**发现**: App 原型体验有**两套独立实现**

#### 方案 A：模板文件（推荐 ✅）
```python
# 第 2847-2880 行
@app.route('/onboarding')
def onboarding():
    return render_template('onboarding.html')

@app.route('/auth')
def app_auth():
    return render_template('auth.html')

@app.route('/personality_customization')
def personality_customization():
    return render_template('personality_customization.html')

@app.route('/app_main')
def app_main():
    return render_template('app_main.html')
```

**优点**:
- ✅ 代码结构清晰
- ✅ 易于维护和扩展
- ✅ 支持热重载
- ✅ 符合 Flask 最佳实践

#### 方案 B：内联 HTML（不推荐 ⚠️）
```python
# 第 2943-3293 行（350 行代码）
@app.route('/app')
def app_experience():
    return '''<html>...内联 HTML...</html>'''

@app.route('/app/onboarding')
def app_onboarding():
    # API 处理

@app.route('/app/customization')
def app_customization():
    # API 处理
```

**缺点**:
- ❌ 代码冗长（350+ 行）
- ❌ 难以维护
- ❌ 无法利用模板继承
- ❌ 与方案 A 功能重复

## ⚠️ 四、发现的问题

### 4.1 主要问题

**问题 1**: 双重实现导致复杂度增加

- 两套独立的 App 体验实现
- 占用 350+ 行代码
- 增加维护成本

**影响**: 
- 开发者可能混淆使用哪个入口
- 代码审查困难
- Bug 修复需要同步两处

---

### 4.2 次要问题

**问题 2**: 用户体验循环

现象：
```
/app_main → 检查 onboarding_completed=False → 重定向到 /onboarding
```

原因：
- 用户尚未填写名字和 AI 分身名字
- `onboarding_completed` 标志为 False

**解决方案**:
1. 在 `/onboarding` 页面添加表单提交
2. 提交后设置 `onboarding_completed=True`
3. 即可正常进入 `/app_main`

#### 建议 2: 完善 Onboarding 流程

**当前缺失**:

- 无法设置 `onboarding_completed=True`

**修复方案**:
在 `templates/onboarding.html` 中添加：
```javascript
function continueToAuth() {
    const userName = document.getElementById('userName').value;
    const aiName = document.getElementById('aiName').value;
    
    fetch('/api/save_avatar_info', {
        method: 'POST',
        headers: {'Content-Type': 'application/json'},
        body: JSON.stringify({
            user_name: userName,
            ai_name: aiName,
            avatar_style: 'human'
        })
    })
    .then(res => res.json())
    .then(data => {
        if (data.success) {
            window.location.href = '/auth';
        }
    });
}
```
