# 用户数据持久化修复完成

## ✅ 问题已解决

**问题描述**：用户每次退出后重新进入，都会回到头像设置界面，并且显示的是第一次上传的图片。

**根本原因**：
1. 用户 ID 仅存储在 Flask session 中
2. session 在浏览器关闭后会丢失
3. 导致每次都生成新的 user_id，无法加载之前的头像数据

## 🔧 修复方案

采用 **Cookie 持久化用户 ID** 方案，确保用户数据永久保存。

### 修改的文件

**1. app.py**

**修改 1**: 添加 `@app.before_request` 从 Cookie 加载用户 ID
```python
@app.before_request
def load_user_from_cookie():
    """从 Cookie 加载用户 ID（如果存在）"""
    user_id = request.cookies.get('user_id')
    if user_id and 'user_id' not in session:
        session['user_id'] = user_id
        print(f"✅ 从 Cookie 加载用户 ID: {user_id[:8]}...")
```

**修改 2**: 在 `api_profile_setup` 中设置 Cookie
```python
# 获取或创建用户 ID
user_id = get_or_create_user_id()

# ... 保存用户数据 ...

# 设置 Cookie 持久化用户 ID
response = make_response(jsonify({'success': True, 'user_id': user_id}))
response.set_cookie('user_id', user_id, max_age=31536000)  # 1 年
return response
```

### 工作原理

1. **首次设置头像**
   - 用户完成头像设置
   - 后端生成 user_id 并保存到 Cookie（有效期 1 年）
   - 同时保存到数据库和 localStorage

2. **关闭浏览器后重新打开**
   - 浏览器自动发送 Cookie（包含 user_id）
   - `@app.before_request` 从 Cookie 读取 user_id
   - 恢复到 session 中
   - 从数据库加载用户之前保存的头像数据

3. **后续访问**
   - Cookie 持续有效（1 年）
   - 用户无需重新设置头像
   - 直接显示之前保存的头像

## 📝 技术细节

### Cookie vs Session

| 特性 | Session | Cookie（本次修复） |
|------|---------|-------------------|
| 存储位置 | 服务器内存 | 浏览器 |
| 有效期 | 浏览器会话期间 | 1 年（可配置） |
| 自动发送 | ❌ | ✅ |
| 持久化 | ❌ | ✅ |

### 为什么选择 Cookie？

1. **简单易用** - 浏览器自动管理
2. **持久化** - 关闭浏览器不丢失
3. **自动发送** - 每次请求自动携带
4. **安全性** - HttpOnly 标志防止 XSS 攻击（未来可添加）

### 数据存储结构

**Cookie**:
```

```

**数据库** (user_data/abc12345.json):
```json
{

  "created_at": "2026-03-20T10:00:00",
  "go_gender": "male",
```

## ⚠️ 注意事项

### 1. Cookie 有效期
- 当前设置为 **1 年**
- 如需修改，调整 `max_age` 参数：
  ```python
  max_age=2592000  # 30 天
  max_age=604800   # 7 天
  max_age=86400    # 1 天
  ```

### 2. 多设备问题
- Cookie 是**单设备**的
- 在不同设备上会生成不同的 user_id
- 解决方案：未来实现账号系统

### 3. 隐私模式
- 浏览器隐私模式可能会阻止 Cookie
- 用户每次打开都会是新用户
- 这是正常行为

### 4. 清除 Cookie
用户清除 Cookie 后：
- 会回到初始状态
- 需要重新设置头像
- 这是预期行为
