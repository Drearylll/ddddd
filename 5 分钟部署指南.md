# 🚀 5 分钟部署 Go In 到公网

## ⏱️ 时间线

- **0:00 - 1:00** - 上传到 GitHub
- **1:00 - 3:00** - Vercel 部署
- **3:00 - 5:00** - 测试并分享链接

---

## 📝 步骤详解

### 步骤 1：上传到 GitHub（1 分钟）

#### 1.1 初始化 Git

打开命令行（CMD 或 PowerShell），进入项目目录：

```bash
cd "c:\Users\hn\Desktop\桌面的东西\GOIN2"
```

执行：

```bash
git init
git add .
git commit -m "Initial commit"
```

#### 1.2 创建 GitHub 仓库

1. 访问 https://github.com
2. 点击右上角 "+" → "New repository"
3. Repository name: `goin`
4. 选择 "Public"
5. 点击 "Create repository"

#### 1.3 推送代码

复制 GitHub 提示的命令，例如：

```bash
git remote add origin https://github.com/你的用户名/goin.git
git branch -M main
git push -u origin main
```

✅ **完成标志：** GitHub 仓库页面能看到代码

---

### 步骤 2：Vercel 部署（2 分钟）

#### 2.1 访问 Vercel

1. 访问 https://vercel.com
2. 点击 "Continue with GitHub" 登录

#### 2.2 导入项目

1. 点击 "Add New..." → "Project"
2. 在 "Import Git Repository" 标签下找到你的 `goin` 仓库
3. 点击 "Import"

#### 2.3 部署

1. 保持默认设置（不需要修改任何东西）
2. 点击 "Deploy"
3. 等待 30-60 秒

✅ **完成标志：** 看到 "Congratulations!" 提示

---

### 步骤 3：获取公网链接（30 秒）

#### 3.1 查看链接

部署完成后，页面顶部会显示：

```
https://goin-xxx.vercel.app
```

这个就是你的公网链接！

#### 3.2 测试链接

1. 复制链接
2. 在手机和电脑上分别打开
3. 确认能正常显示内容

✅ **完成标志：** 看到 3-5 条内容，时间标签正常

## 🔧 常见问题

### Q1: 部署失败怎么办？

**A:** 检查以下几点：
1. GitHub 仓库是否是 Public
2. 是否有 `requirements.txt` 文件
3. 是否有 `vercel.json` 文件

**解决：**
```bash
# 重新推送代码
git push origin main
```

然后在 Vercel 点击 "Redeploy"

---

### Q2: 看到欢迎页而不是内容流？

**A:** 代码未正确修改

**解决：**
检查 `app.py` 第 988-998 行是否是：

```python
if not user_data.get('preferences'):
    # 自动生成默认偏好（随机）
    default_prefs = {
        'style': random.choice(['quiet', 'social']),
        'social': random.choice(['alone', 'crowd']),
        'time': random.choice(['day', 'night']),
    }
    save_user_data('preferences', default_prefs)
```

然后重新部署：
```bash
git add .
git commit -m "Fix onboarding issue"
git push origin main
```

---

### Q3: 只生成 2 条内容？

**A:** 代码未更新

**解决：**
检查 `app.py` 第 1075-1077 行：

```python
num_posts_to_generate = random.randint(3, 5)
print(f"[LIFE] First time generation: {num_posts_to_generate} posts (initial experience)")
```

然后重新部署。

---

### Q4: 图片无法加载？

**A:** 静态文件路径问题

**解决：**
确保 `static/images/` 文件夹存在且有图片文件。
