# Go In App 行业级升级包 v3（可直接投喂通义千码）

## 目标
- 打造“微信级清爽可读 + 抖音级内容突出”的社交内容体验。
- 形成 Go In 独立品牌风格：自然低饱和底色 + 精致品牌强调色。
- 保持模块化，可独立迭代页面、动效、交互、性能策略。

## 当前已落地文件
- `static/css/goin_brand_v3.css`：全局品牌设计 Tokens + 通用组件样式 + 动效规范。
- `static/js/goin_brand_v3.js`：统一交互脚本（Tab 切换、点赞反馈、手势滑动、Toast、骨架屏控制）。
- `templates/app_main.html`：行业风格主界面（顶部导航 + 四 Tab + 卡片/列表混合 + 手势切换 + 占位渐入）。

## 设计系统（Tokens）
### 颜色
- 底色：`#f6f6f4` / `#ffffff` / `#f1f1ef`
- 文字：`#1f1f1f` / `#5f6368` / `#84888e`
- 品牌色：`#17c964`（主强调） + `#12a150`（深强调）
- 高亮备用：`#ff2f63`
- 品牌渐变：`linear-gradient(135deg, #14c96a 0%, #39c7ff 100%)`

### 字体与层级
- 正文：14-16px，行高 1.5-1.6
- 卡片标题：20px
- 顶栏 Logo：18px
- 辅助信息：12px

### 圆角与阴影
- 卡片圆角：22px
- 中按钮圆角：12px
- 微阴影：`0 2px 8px rgba(18,24,40,.06)`
- 卡片阴影：`0 8px 20px rgba(18,24,40,.08)`

### 动效
- 快速反馈：160ms
- 过渡动效：260ms
- 进入动画：420ms
- 缓动：`cubic-bezier(0.2, 0.8, 0.2, 1)`

## 模块结构（可独立迭代）
### 1) 顶部导航
- 结构：品牌 Logo + 搜索入口 + 功能集合入口
- 特征：半透明吸顶、轻磨砂、极简层级

### 2) 底部导航
- 4 Tab：社区 / 聊天 / 朋友圈 / 我的
- 支持点击切换 + 左右滑动手势切换
- Tab 激活态仅使用品牌色，保持克制

### 3) 社区信息流（卡片 + 列表混合）
- 内容类型 Chip：图像 / 视频 / 音频 / 小说
- 卡片布局：媒体区 + 标签 + 简述 + 互动区
- 操作：点赞微动效、创作入口、存档入口

### 4) 聊天模块
- 支持真人/AI代理模式切换
- 代聊记录回看：保留/修改/删除/转述
- 反馈及时，尽量一步操作

### 5) AI 朋友圈
- 自动生成定位打卡内容
- 附近场景推荐（景点/餐厅/书店）
- 现实拍照解锁入口

### 6) 我的
- 用于设置、反馈、规范透出、版本升级入口

## 交互与反馈规范
- 点击：按钮按压缩放（0.95）
- 点赞：icon Pop 动画（300ms）
- 切页：Fade + TranslateY（420ms）
- 加载：骨架屏渐变流光
- 提示：底部 Toast，1.2s 自动消失

## 手势策略
- 横向滑动切 Tab：阈值 48px
- 纵向滚动优先内容浏览，避免手势冲突
- 手势误触保护：横向位移需大于纵向位移

## 性能策略（前端原型阶段）
- 首屏先渲染骨架，内容填充后淡入
- 减少重排：卡片容器固定结构
- 过渡尽量使用 transform/opacity
- JS 交互集中在 `goin_brand_v3.js`，避免散落逻辑

## 下一轮建议（按优先级）
1. 将 `onboarding/auth/personality_customization` 也切换到 `goin_brand_v3.css` 体系。
2. 新增“沉浸视频流”页面（参考抖音单列沉浸式）。
3. 将 `goin_brand_v3.js` 拆为 `store + interaction + ui-feedback` 三模块。
4. 增加无障碍参数（文本缩放、对比度模式、减少动效模式）。
5. 引入真实图片/视频资源，验证长列表滚动帧率。
