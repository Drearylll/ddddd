## 🎯 核心功能说明

### ContentGenerator 类

#### 主要方法

```python
@staticmethod
def generate_ai_life_posts(user_data, num_posts=3, current_time=None):
    """
    AI 自主生活流内容生成
    
    功能：
    1. 根据当前时间段过滤内容
    2. 根据用户人格标签加权
    3. 返回排序后的内容列表
    
    参数：
    - user_data: 用户数据（包含兴趣标签、权重等）
    - num_posts: 生成数量
    - current_time: 当前时间
    
    返回：
    - list: 内容列表，每条包含 location, text, mood, time_type 等
    """
```

#### 私有方法

- `_get_time_period()` - 根据小时数返回时间段（morning/afternoon/night）
- `_get_candidate_contents()` - 返回预设的候选内容池（5 条示例数据）
- `_filter_by_personality()` - 根据时间段和用户标签过滤并排序内容
- `_get_time_type_weights()` - 返回不同时间段的内容类型权重
- `_calculate_personality_weight()` - 计算单条内容的人格权重

---

### UserManager 类

#### 主要方法

```python
@staticmethod
def get_user_data(user_id=None):
    """
    获取用户数据
    
    从 user_data/{user_id}.json 文件加载数据
    支持从 session 自动获取 user_id
    
    返回：
    - dict: 用户数据字典
    """

@staticmethod
def save_user_data(key, value, user_id=None):
    """
    保存用户数据
    
    将数据保存到 user_data/{user_id}.json
    自动创建 user_data 目录
    
    参数：
    - key: 数据键
    - value: 数据值
    - user_id: 用户 ID（可选）
    """

@staticmethod
def get_or_create_user_id():
    """
    获取或创建用户 ID
    
    如果 session 中没有 user_id，则生成新的 UUID
    自动初始化 created_at 和 user_id 字段
    
    返回：
    - str: 8 位用户 ID
    """
```

## 🔧 使用示例

### 在 app.py 中使用服务层

#### 示例 1：在 /app_main 中使用

```python
from services import ContentGenerator, UserManager

@app.route('/app_main')
def app_main():
    """主界面 - 使用服务层生成内容"""
    
    # 获取用户数据
    user_data = UserManager.get_user_data()
    
    # 检查是否完成引导
    onboarding_completed = user_data.get('onboarding_completed', False)
    if not onboarding_completed:
        return redirect(url_for('onboarding'))
    
    # 使用服务层生成 AI 生活流内容
    posts = ContentGenerator.generate_ai_life_posts(
        user_data=user_data,
        num_posts=5
    )
    
    return render_template('app_main.html', posts=posts)
```

## 🎯 架构优势

### 分层架构

```
┌─────────────────────────────────────┐
│         表现层 (templates/)          │
│  - welcome_premium.html             │
│  - app_main.html                    │
│  - personality_customization.html   │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│         路由层 (app.py)              │
│  - @app.route('/')                  │
│  - @app.route('/app_main')          │
│  - @app.route('/api/*')             │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│         服务层 (services/)           │
│  - ContentGenerator                 │
│  - UserManager                      │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│         数据层 (user_data/)          │
│  - {user_id}.json                   │
└─────────────────────────────────────┘
```

### 职责分离

| 层级 | 职责 | 文件 |
|------|------|------|
| 表现层 | UI 展示、用户交互 | templates/*.html |
| 路由层 | URL 路由、请求处理 | app.py |
| 服务层 | 业务逻辑、数据处理 | services/*.py |
| 数据层 | 数据持久化 | user_data/*.json |
