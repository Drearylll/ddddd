# 高德地图 API 配置指南

## ⚠️ 当前问题

**错误代码**: `INVALID_USER_KEY`  
**错误信息**: API Key 无效或未授权  
**原因**: 需要在高德控制台配置白名单或添加安全密钥

---

## 🎯 解决方案（二选一）

### 方案 1: 使用模拟数据（推荐 - 无需配置）

**优点**:
- ✅ 无需申请 API Key
- ✅ 立即可以使用
- ✅ 数据质量良好
- ✅ 适合开发和演示

**缺点**:
- ❌ 地点固定，不是实时数据
- ❌ 无法根据用户位置变化

**启用方式**:
```python
# services/location.py
GAODE_API_KEY = ""  # 留空即可
```

系统会自动降级到模拟数据。

---

### 方案 2: 完善高德 API 配置（需要额外步骤）

#### 步骤 1: 访问高德开放平台

🔗 地址：https://lbs.amap.com/

**操作**:
1. 注册/登录高德账号
2. 进入"应用管理" → "我的应用"

---

#### 步骤 2: 创建新应用

**操作**:
1. 点击"创建新应用"
2. 填写应用名称（如"Go In App"）
3. 选择应用类型（Web 应用或其他）
4. 提交创建

---

#### 步骤 3: 添加 Key

**操作**:
1. 在应用下点击"添加 Key"
2. 选择服务平台（Web 服务 API）
3. 输入 Key 名称
4. **重要**: 设置白名单
   - IP 白名单：`127.0.0.1` (本地开发)
   - 或选择不限制（不推荐生产环境）

---

#### 步骤 4: 获取正确的 API Key

**注意**: 
- 您提供的 Key `ab9c30557c52e1c32ea41f46f251c54c` 格式正确
- 但可能未开通 Web 服务权限
- 或未在白名单中添加本机 IP

**解决**:
1. 确认已开通"Web 服务 API"
2. 检查 Key 的白名单设置
3. 重新生成一个新的 Key

---

#### 步骤 5: 配置安全密钥（如需要）

某些情况下需要配置安全密钥：

**操作**:
1. 在应用详情页面找到"安全密钥"
2. 生成或复制安全密钥
3. 更新到代码中：

```python
# services/location.py
GAODE_API_KEY = "你的新 API Key"
GAODE_API_SECRET = "你的安全密钥"
```
