# 轻习惯系统机制说明

## 一、核心目标

让 AI 逐渐呈现稳定偏好，形成类似真实人的生活习惯。

**用户体验目标：**
> "这个人，好像一直都是这样生活的"

---

## 二、实现方式

### 1. 初始化"生活倾向"（每个用户一个）

每个新用户首次访问时，随机分配 1-2 个生活倾向：

**倾向类型（5 种）：**

| 倾向 | 说明 | 偏好地点 | 偏好时间 | 权重提升 |
|------|------|----------|----------|----------|
| **咖啡倾向** | 更容易生成咖啡店相关，出现"坐""停留" | 张江咖啡馆、IFC 商场外、金桥路街边 | day | 2.0x |
| **夜晚倾向** | 更容易生成夜晚时间内容 | 陆家嘴滨江步道、世纪大道地铁站口、浦东美术馆 | night | 2.0x |
| **散步倾向** | 更容易生成步行、移动相关内容 | 陆家嘴滨江步道、前滩友城公园、世纪公园 | any | 1.8x |
| **安静倾向** | 更容易生成安静、独处内容 | 张江咖啡馆、浦东美术馆、世纪公园 | any | 2.0x |
| **购物倾向** | 更容易生成商场、商业区内容 | IFC 商场外、正大广场门口、前滩太古里、金桥国际广场 | day | 1.5x |

**初始化逻辑：**
```python
def init_life_traits():
    """随机分配 1-2 个生活倾向"""
    if not user_data.get('life_traits'):
        num_traits = random.randint(1, 2)  # 随机 1-2 个
        all_traits = list(LIFE_TRAITS.keys())
        selected_traits = random.sample(all_traits, num_traits)
```

**示例：**
- 用户 A：`['coffee', 'quiet']` - 咖啡 + 安静倾向
- 用户 B：`['night', 'walking']` - 夜晚 + 散步倾向
- 用户 C：`['shopping']` - 仅购物倾向

### 2. 内容生成逻辑

根据倾向提高相关内容概率：

#### 地点选择权重算法

```python
def select_location_with_traits(candidate_locations):
    """根据用户的生活倾向选择地点（提高相关地点概率）"""
    
    # 计算每个地点的权重
    location_scores = []
    for location in candidate_locations:
        base_score = 1.0
        
        # 检查该地点是否符合用户的生活倾向
        for trait_key in user_traits:
            trait = LIFE_TRAITS.get(trait_key)
            if trait and location in trait['preferred_locations']:
                base_score *= trait['weight_boost']
        
        location_scores.append((location, base_score))
    
    # 根据权重随机选择（权重越高越可能被选中）
    total_score = sum(score for _, score in location_scores)
    rand_value = random.uniform(0, total_score)
    
    # 轮盘赌选择
    cumulative_score = 0
    for location, score in location_scores:
        cumulative_score += score
        if rand_value <= cumulative_score:
            return location
```

**示例场景：**

用户 A 有 `['coffee', 'quiet']` 倾向，候选地点：
- IFC 商场外：基础 1.0 × 咖啡 2.0 = **2.0 分**
- 张江咖啡馆：基础 1.0 × 咖啡 2.0 × 安静 2.0 = **4.0 分**
- 世纪大道：基础 1.0 = **1.0 分**

结果：张江咖啡馆被选中的概率是世纪大道的 4 倍！

### 3. 地点重复机制

**规则：**
- 20 条内容中，允许 2-4 个地点重复出现
- 优先重复"已出现过的地点"
- 20% 概率触发重复

**实现逻辑：**
```python
# 获取用户已访问的地点记录
visited_locations = user_data.get('visited_locations', [])

# 20 条内容中，允许 2-4 个地点重复出现
max_repeat_locations = random.randint(2, 4)
repeat_count = 0

for i in range(num_posts_to_generate):
    # 基于路径连续性选择地点
    new_location = get_next_location(last_location)
    
    # 检查是否应该重复使用已访问过的地点
    if repeat_count < max_repeat_locations and visited_locations:
        # 20% 概率重复使用已访问过的地点
        if random.random() < 0.2:
            repeated_location = random.choice(visited_locations[-10:])
            new_location = repeated_location
            repeat_count += 1
    
    # 更新已访问地点记录
    if new_location not in visited_locations:
        visited_locations.append(new_location)
        # 保留最近 20 个地点
        if len(visited_locations) > 20:
            visited_locations = visited_locations[-20:]
```

**效果示例：**

```
第 2 条：金科路商业街（新地点）

第 4 条：张江地铁站（新地点）
第 5 条：张江咖啡馆（再次重复！）
```

## 八、限制条件

### ❌ 不做的事情

1. **不要让用户看到"设定"** - 生活倾向对用户不可见
2. **不要明显标签化** - 不显示"你有咖啡倾向"等提示
3. **保持自然随机** - 仍然是概率选择，不是强制

---

### ✅ 要做的事情

1. **保持隐蔽性** - 用户只能感受到习惯，看不到数据
2. **保持随机性** - 权重提升是概率性的，不是绝对的
3. **保持一致性** - 同一个用户的生活倾向不会改变

---

## 九、设计原则

### 1. 平行自我原则

内容必须表现为"一个可能的用户版本"在持续生活：
- ✅ 稳定的生活习惯（总是去同一家咖啡馆）
- ✅ 一致的行为模式（喜欢坐下来发呆）
- ✅ 可预测的偏好（夜晚出来散步）

---

### 2. 真实世界映射

真实的人都有习惯：
- ✅ 会重复去喜欢的地方
- ✅ 有固定的行为模式
- ✅ 偏好某些时间段活动

---

### 3. 不可控体验

用户不能控制或选择：
- ✅ 生活倾向是随机分配的
- ✅ 用户无法看到或修改
- ✅ 习惯是自然形成的
