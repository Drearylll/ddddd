## 🎯 发现的问题

### 1. 双重体验路径 ⭐⭐⭐⭐⭐

**现状**：
```
路径 A: / → /feed (Web 轻量版，约 1500 行代码)
路径 B: / → /welcome → /personality_customization → /app_main (App 完整版)
```

**问题**：
- 两套独立的用户流程
- 内容生成逻辑分散
- 用户体验不一致

**建议**：统一到路径 B（行业顶级设计）

---

### 2. API 端点分散 ⭐⭐⭐

**当前 API**（9 个）：
```python
/save_preferences              # 表单提交
/api/save_avatar_info         # JSON 提交
/api/answer_question          # AI 回答
/api/toggle_agent_mode        # 切换模式
/api/agent_respond            # Agent 响应
/api/review_agent_responses   # 查看历史
/api/feedback_agent_response  # 反馈
/api/next_post                # 下一条内容
/api/save_insight             # 保存洞察
```

**问题**：
- 命名风格不统一
- 缺少文档
- 难以维护

**建议**：统一为 RESTful 风格

---

### 3. 用户数据管理混乱 ⭐⭐⭐

**三重存储**：
```javascript
// 前端 localStorage
localStorage.setItem('goIn_userName', userName);
```

```python
# 后端 Session
session['user_name'] = user_name
```

```python
# 文件存储
save_user_data('preferences', prefs)
```

**问题**：
- 数据同步困难
- 调试复杂
- 可能出现不一致

**建议**：统一使用 Session + 后端存储

#### 任务 5：数据库升级

**从文件存储升级到 SQLite**：

```python
# models/database.py
import sqlite3
from datetime import datetime
import json

class Database:
    def __init__(self, db_path='goin.db'):
        self.conn = sqlite3.connect(db_path)
        self.init_db()
    
    def init_db(self):
        c = self.conn.cursor()
        
        # 用户表
        c.execute('''
            CREATE TABLE IF NOT EXISTS users (
                id TEXT PRIMARY KEY,
                data TEXT NOT NULL,
                created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
                updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
            )
        ''')
        
        # 内容表
        c.execute('''
            CREATE TABLE IF NOT EXISTS posts (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                user_id TEXT NOT NULL,
                content TEXT NOT NULL,
                location TEXT,
                timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
                FOREIGN KEY (user_id) REFERENCES users(id)
            )
        ''')
        
        self.conn.commit()
    
    def save_user(self, user_id, data):
        c = self.conn.cursor()
        c.execute('''
            INSERT OR REPLACE INTO users (id, data, updated_at)
            VALUES (?, ?, ?)
        ''', (user_id, json.dumps(data), datetime.now()))
        self.conn.commit()
    
    def get_user(self, user_id):
        c = self.conn.cursor()
        c.execute('SELECT data FROM users WHERE id = ?', (user_id,))
        row = c.fetchone()
        return json.loads(row[0]) if row else {}
```

**好处**：
- ✅ 数据结构化
- ✅ 查询效率高
- ✅ 支持复杂关系

## 🎯 关键决策点

### 决策 1：是否保留 /feed 路由？

**选项 A：删除** ⭐⭐⭐⭐⭐（推荐）
- 专注做精品的 App 体验
- 减少维护负担
- 统一用户体验

**选项 B：保留并重构** ⭐⭐⭐
- 提供轻量级入口
- 可以作为引流工具
- 需要额外维护成本

**建议**：选择 **选项 A**

**理由**：
1. 产品定位是"一个会自己发生的世界"
2. 需要完整的用户体验才能传达这个理念
3. Web 轻量版虽然门槛低，但无法体现产品核心价值
4. 资源应该集中在打造精品上

**选项 B：稍后升级** ⭐⭐⭐
